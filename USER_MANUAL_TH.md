# 🛡️ คู่มือการใช้งาน: ระบบขออนุมัติเข้าใช้งานไฟล์วอลล์ (User Manual)

ระบบนี้เป็นช่องทางบริการตนเอง (Self-Service) เพื่อขอสิทธิ์เข้าใช้งานไฟล์วอลล์ชั่วคราว (เช่น RDP) บนอุปกรณ์ **FortiGate** โดยระบบจะทำการสร้างนโยบายความปลอดภัยและกำหนดวันหมดอายุให้โดยอัตโนมัติ

---

## 1. การเข้าสู่ระบบ (Authentication)

ระบบใช้บัญชี **Corporate LDAP** (บัญชีเดียวกับคอมพิวเตอร์บริษัท) ในการตรวจสอบสิทธิ์:

1.  กรอก **Username** (เช่น `natapol.t`) และรหัสผ่าน
2.  หากสำเร็จ ระบบจะจัดเก็บ Session Token ไว้ในบราวเซอร์
3.  **หมายเหตุ:** หากเข้าสู่ระบบไม่ได้ ให้ตรวจสอบว่าบราวเซอร์เปิดใช้งาน Cookie หรือไม่

## 2. ขั้นตอนการขอสิทธิ์ (How to Request)

### ส่วนที่ 1: ข้อมูลเอกสาร (Document Info)
ระบุชื่อเอกสารหรือโปรเจกต์ (**Document Name**) เพื่อใช้ในการอ้างอิงและระบุตัวตนของกฎบนไฟล์วอลล์

### ส่วนที่ 2: รายละเอียดผู้ใช้งาน (IP Address)
กรอกชื่อและหมายเลข IP (เช่น `10.10.x.x`)
- สามารถคลิกปุ่ม **"+ เพิ่มผู้ใช้งาน"** เพื่อขอสิทธิ์ให้เพื่อนร่วมทีมหลายคนพร้อมกันในคำขอเดียวได้

### ส่วนที่ 3: กำหนดเวลา (Expiry)
เลือกวันสุดท้ายที่ต้องการใช้งาน
- ระบบจะยกเลิกสิทธิ์โดยอัตโนมัติในเวลา **23:59 น.** ของวันที่คุณเลือก

## 3. การทำงานของระบบ (Behind the Scenes)

เพื่อให้ผู้ดูแลระบบ (Admin) มั่นใจในความปลอดภัย ระบบมีการทำงานดังนี้:

*   **Address Object:** ระบบสร้างวัตถุ IP ในชื่อ `ADDR_[IP]` ให้โดยอัตโนมัติหากยังไม่มีในระบบ
*   **Policy Merging:** หากมีการขอสิทธิ์ในวันเดียวกัน ระบบจะรวมรายชื่อ IP ไว้ใน Policy เดียวกันเพื่อลดความซับซ้อนของกฎบนไฟล์วอลล์
*   **Rule Precedence:** นโยบายใหม่จะถูกย้ายไปอยู่ก่อนหน้า **Rule ID 285** (ซึ่งมักจะเป็นกฎ Block) เพื่อให้ใช้งานได้ทันที
*   **Notifications:** ระบบจะส่งอีเมลยืนยันผลการดำเนินการผ่าน SMTP ทันทีที่ตั้งค่าบน FortiGate สำเร็จ

## 4. การแก้ปัญหาเบื้องต้น (Troubleshooting)

| ปัญหา (Problem) | สาเหตุ (Cause) | วิธีแก้ไข (Solution) |
| :--- | :--- | :--- |
| **หน้าจอวนลูป Login** | Cookie/SSL Policy | เข้าใช้งานผ่าน HTTPS หรือแจ้ง Admin ให้ตั้งค่า `COOKIE_SECURE=false` |
| **IP Address ไม่ถูกต้อง** | พิมพ์ผิดรูปแบบ | ตรวจสอบให้แน่ใจว่ากรอกในรูปแบบ X.X.X.X (เช่น 10.0.0.1) |
| **ติดต่อเซิร์ฟเวอร์ไม่ได้** | Backend Down | ตรวจสอบสถานะ Docker Container บนเซิร์ฟเวอร์ |
| **ไม่ได้รับอีเมลยืนยัน** | SMTP Config | ตรวจสอบการตั้งค่า SMTP ในไฟล์ `.env` ของระบบ |
