คู่มือการใช้งาน (User Manual)

KCE Firewall Access Request System

1
ภาพรวมระบบ (Overview)

ระบบนี้เป็นช่องทางบริการตนเอง (Self-Service) เพื่อขอสิทธิ์เข้าใช้งานไฟล์วอลล์ชั่วคราว (เช่น RDP) บนอุปกรณ์ FortiGate โดยระบบจะทำการสร้างนโยบายความปลอดภัยและกำหนดวันหมดอายุให้โดยอัตโนมัติ เพื่อลดขั้นตอนการส่ง Ticket แบบปกติ

2
การเข้าสู่ระบบ (Authentication)

ระบบใช้บัญชี Corporate LDAP ในการตรวจสอบสิทธิ์:

  1. กรอก Username (เช่น natapol.t) และรหัสผ่าน
  2. หากสำเร็จ ระบบจะจัดเก็บ Session Token ไว้ในบราวเซอร์
  3. *กรณีเข้าสู่ระบบไม่ได้: ตรวจสอบว่าบราวเซอร์เปิดใช้งาน Cookie หรือไม่

3
ขั้นตอนการขอสิทธิ์ (How to Request)

ส่วนที่ 1: ข้อมูลเอกสาร

ระบุชื่อเอกสารหรือโปรเจกต์ (Document Name) เพื่อใช้ในการอ้างอิงบนไฟล์วอลล์

ส่วนที่ 2: รายละเอียดผู้ใช้งาน (IP Address)

กรอกชื่อและหมายเลข IP (เช่น 10.10.x.x) สามารถคลิกปุ่ม "+ เพิ่ม" เพื่อขอสิทธิ์ให้หลายคนพร้อมกันได้

ส่วนที่ 3: กำหนดเวลา (Expiry)

เลือกวันสุดท้ายที่ต้องการใช้งาน ระบบจะยกเลิกสิทธิ์อัตโนมัติในเวลา 23:59 น. ของวันนั้น

4
การทำงานของระบบ (Behind the Scenes)

Address Object

ระบบสร้างวัตถุ IP ในชื่อ ADDR_[IP] ให้อัตโนมัติ

Policy Merging

หากขอสิทธิ์วันเดียวกัน ระบบจะรวมไว้ใน Policy เดียวกันเพื่อความเป็นระเบียบ

Rule Precedence

นโยบายใหม่จะถูกย้ายไปอยู่ก่อนหน้า Rule ID 285 (Block Rule) เสมอ

Notifications

ส่งอีเมลยืนยันผลการดำเนินการผ่าน SMTP ทันทีที่ตั้งค่าสำเร็จ

5
การแก้ปัญหาเบื้องต้น (Troubleshooting)

ปัญหา (Problem) สาเหตุ (Cause) วิธีแก้ไข (Solution)
หน้าจอวนลูป Login Cookie/SSL Policy เข้าใช้งานผ่าน HTTPS หรือตั้งค่า COOKIE_SECURE=false
IP Address ไม่ถูกต้อง Format Error กรอกในรูปแบบ X.X.X.X (เช่น 10.0.0.1)
ติดต่อเซิร์ฟเวอร์ไม่ได้ Backend Down ตรวจสอบสถานะ Docker Container บนเซิร์ฟเวอร์
ไม่ได้รับอีเมลยืนยัน SMTP Config ตรวจสอบ SMTP Credentials ในไฟล์ .env
